To have rtdl build your data lake on Azure Blob Storage, you need to:
  1.  Login to your [Azure Portal](https://portal.azure.com/).
  2.  Create a Storage Account of type General-purpose v2 account.
  3.  Choose your storage account and under Blob Service , click on `Containers`.
  4.  Create a new container. This is where all your data lake files will be stored.
  5.  Under Settings click on `Access Keys`.
      * Save the Storage Account Name and Access Key for use in configuring your stream in rtdl.
  6.  Configure your stream in rtdl.  
      Send a call to the API at http://localhost:8080/ingest.
      * Example `createStream` call body for creating a data lake on Azure Blob Storage.  
        ```
        {
          "active": true,
          "message_type": "test-msg-azure",
          "file_store_type_id": 4,
          "bucket_name": "testBucketAzure",
          "folder_name": "testFolderAzure",
          "partition_time_id": 1,
          "compression_type_id": 1,
          "azure_storage_account_name": "[azure_storage_account_name]",
          "azure_storage_access_key": "[azure_storage_access_key]"
        }
        ```
      * Example `createStream` curl call for creating a data lake on Azure Blob Storage.  
        ```
        curl --location --request POST 'http://localhost:80/createStream' \
        --header 'Content-Type: application/json' \
        --data-raw '{
          "active": true,
          "message_type": "test-msg-azure",
          "file_store_type_id": 4,
          "bucket_name": "testBucketAzure",
          "folder_name": "testFolderAzure",
          "partition_time_id": 1,
          "compression_type_id": 1,
          "azure_storage_account_name": "[azure_storage_account_name]",
          "azure_storage_access_key": "[azure_storage_access_key]"
        }'
        ```
